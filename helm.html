Helm

The package manager for Kubernetes — manage, template, and deploy applications with charts.

Core Concepts

📦 Chart
A package of pre-configured Kubernetes resources. Contains templates, default values, and metadata.
🚀 Release
An instance of a chart running in a cluster. Each install creates a new release with its own name and revision history.
📂 Repository
A server that hosts packaged charts. Add repos to discover and install community charts (e.g., Bitnami, stable).
📝 Values
Configuration parameters that customize chart behavior. Override defaults via -f values.yaml or --set key=value.

Chart Structure

my-chart/
├── Chart.yaml          # Chart metadata (name, version, appVersion)
├── Chart.lock          # Dependency lock file
├── values.yaml         # Default configuration values
├── values.schema.json  # JSON schema for values validation
├── templates/
│   ├── _helpers.tpl    # Reusable template partials
│   ├── deployment.yaml # Kubernetes Deployment
│   ├── service.yaml    # Kubernetes Service
│   ├── ingress.yaml    # Ingress resource
│   ├── hpa.yaml        # Horizontal Pod Autoscaler
│   ├── configmap.yaml  # ConfigMap
│   ├── secret.yaml     # Secret
│   ├── NOTES.txt       # Post-install usage notes
│   └── tests/
│       └── test-connection.yaml
├── charts/             # Sub-chart dependencies
└── .helmignore         # Files to exclude from packaging

Templating Basics

{{ .Values.* }}
Access values from values.yaml: 
image: {{ .Values.image.repository }}:{{ .Values.image.tag }}
{{ .Release.* }}
Built-in release info: 
name: {{ .Release.Name }}
namespace: {{ .Release.Namespace }}
revision: {{ .Release.Revision }}
Control Flow
Conditionals and loops: 
{{- if .Values.ingress.enabled }}
  # render ingress
{{- end }}

{{- range .Values.env }}
- name: {{ .name }}
  value: {{ .value | quote }}
{{- end }}
Helper Functions
Define in _helpers.tpl: 
{{- define "mychart.fullname" -}}
{{ .Release.Name }}-{{ .Chart.Name }}
{{- end }}

# Usage:
name: {{ include "mychart.fullname" . }}

Best Practices

✅ Do
  • • Use helm template before installing
  • • Pin chart versions in CI/CD
  • • Use values.schema.json for validation
  • • Namespace your releases
  • • Use --atomic for auto-rollback on failure
❌ Don't
  • • Don't hardcode values in templates
  • • Don't skip helm lint in CI
  • • Don't use latest image tags
  • • Don't store secrets in values.yaml
  • • Don't modify releases manually with kubectl
